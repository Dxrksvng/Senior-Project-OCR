# 🎓 เว็บแอปพลิเคชันสำหรับตรวจสอบคะแนนสอบภาษาอังกฤษเพื่อคุณสมบัติการจบการศึกษา (Web Application for Verifying English Exam Scores)

## 📝 ภาพรวม (Overview)
[cite_start]โปรเจกต์นี้ถูกพัฒนาขึ้นเพื่อแก้ไขปัญหาและลดขั้นตอนการตรวจสอบเอกสารผลคะแนนสอบภาษาอังกฤษ (เช่น KMITL-TEP, TOEIC) สำหรับการยื่นขอจบการศึกษา ซึ่งในปัจจุบันมีกระบวนการที่ต้องทำด้วยมือ (Manual) ทำให้เกิดความล่าช้าและอาจมีข้อผิดพลาดได้ง่าย[cite: 153].

[cite_start]เว็บแอปพลิเคชันนี้จึงนำเทคโนโลยี **Optical Character Recognition (OCR)** เข้ามาช่วยในการอ่านและสกัดข้อมูลจากไฟล์ผลสอบ (รูปภาพ หรือ PDF) ที่นักศึกษาอัปโหลดเข้ามา จากนั้นระบบจะทำการประมวลผลและตรวจสอบคะแนนโดยอัตโนมัติ เพื่อเพิ่มความรวดเร็ว ลดข้อผิดพลาด และอำนวยความสะดวกให้กับทั้งนักศึกษาและเจ้าหน้าที่ของสถาบัน[cite: 153, 156].

## 🎯 เป้าหมายหลักของโปรเจกต์ (Key Objectives)
* [cite_start]**พัฒนาระบบอัตโนมัติ:** สร้างเว็บแอปพลิเคชันที่สามารถรับไฟล์ผลสอบและสกัดข้อมูลคะแนน, ชื่อ-นามสกุล, และข้อมูลสำคัญอื่นๆ ได้โดยอัตโนมัติ[cite: 153].
* [cite_start]**เปรียบเทียบประสิทธิภาพ OCR:** ศึกษาและเปรียบเทียบประสิทธิภาพของ OCR Engine หลายๆ ตัว ได้แก่ **Tesseract OCR, EasyOCR, และ TrOCR** เพื่อคัดเลือกโมเดลที่เหมาะสมที่สุดสำหรับเอกสารผลสอบภาษาอังกฤษ[cite: 154].
* [cite_start]**เพิ่มความแม่นยำ:** ใช้เทคนิค Image Preprocessing เพื่อเพิ่มความคมชัดและลดสัญญาณรบกวนในเอกสารก่อนนำไปสกัดข้อมูลด้วย OCR[cite: 154].
* [cite_start]**ลดระยะเวลาและข้อผิดพลาด:** สร้างระบบที่ช่วยลดขั้นตอนที่ต้องทำด้วยมือ เพิ่มความถูกต้องแม่นยำ และแสดงผลลัพธ์ได้อย่างรวดเร็ว[cite: 156].

## ⚙️ กระบวนการทำงาน (Workflow)
1.  [cite_start]**อัปโหลดไฟล์ (File Upload):** ผู้ใช้งานอัปโหลดไฟล์ผลคะแนนในรูปแบบ `.pdf`, `.png`, หรือ `.jpg` เข้าสู่ระบบ[cite: 154].
2.  **แปลงไฟล์และประมวลผลภาพ (File Conversion & Pre-processing):**
    * หากไฟล์เป็น PDF ระบบจะทำการแปลงเป็นไฟล์รูปภาพความละเอียดสูง (300 DPI).
    * นำภาพเข้าสู่กระบวนการ Pre-processing เพื่อปรับปรุงคุณภาพของภาพ.
3.  **สกัดข้อมูลด้วย OCR (OCR Extraction):**
    * [cite_start]ระบบใช้ OCR Engine ที่ผ่านการคัดเลือกแล้ว (ผลการทดลองชี้ว่า **EasyOCR** ให้ความแม่นยำสูงสุด [cite: 155]) ในการสกัดข้อความและตัวเลขจากบริเวณที่กำหนดในใบผลคะแนน.
    * มีการเปรียบเทียบแนวทางหลายรูปแบบ เช่น การใช้ OCR โดยตรง (Pure OCR) และการใช้เทคนิคตรวจจับกล่องข้อความ (Boxing OCR) เพื่อให้ได้ผลลัพธ์ที่ดีที่สุด.
4.  **ตรวจสอบและแสดงผล (Verification & Display):**
    * ข้อมูลที่สกัดได้จะถูกนำไปเปรียบเทียบกับเกณฑ์การจบการศึกษาที่กำหนดไว้.
    * แสดงผลการตรวจสอบให้ผู้ใช้ทราบทันทีผ่านหน้าเว็บแอปพลิเคชัน.

## 🛠️ เทคโนโลยีและเครื่องมือที่ใช้ (Tech Stack)
* **ภาษาโปรแกรม:** Python
* **OCR Engines:**
    * [cite_start]**EasyOCR:** โมเดลหลักที่ใช้ในการดำเนินงานจริง เนื่องจากให้ผลลัพธ์แม่นยำที่สุด[cite: 155, 78].
    * **Tesseract OCR:** ใช้เป็นหนึ่งในโมเดลสำหรับเปรียบเทียบประสิทธิภาพ.
    * [cite_start]**TrOCR (Transformer-based OCR):** โมเดล Transformer ที่นำมาทดลองเพื่อเปรียบเทียบความสามารถในการอ่านข้อความ[cite: 154, 81].
* **ไลบรารีหลัก:**
    * `Pytesseract`, `EasyOCR`, `TrOCR`: สำหรับการทำ OCR.
    * `PDF2Image`, `OpenCV`: สำหรับการจัดการไฟล์ PDF และการทำ Image Processing.
    * `Pandas`, `NumPy`: สำหรับการจัดการและวิเคราะห์ข้อมูล.
    * `Streamlit` / `Flask` (ตัวอย่าง): สำหรับการสร้างเว็บแอปพลิเคชัน.

## 📊 ผลการทดลองและประเมินผล (Evaluation & Results)
จากการเปรียบเทียบประสิทธิภาพด้วยชุดข้อมูลใบรับรองคะแนนจริง พบว่า:
* แนวทาง **Boxing OCR** (การตรวจจับกล่องข้อความก่อนสกัด) ให้ผลลัพธ์ที่แม่นยำกว่าการทำ Pure OCR อย่างมีนัยสำคัญในทุกโมเดล.
* **EasyOCR** และ **Tesseract** เมื่อใช้กับแนวทาง Boxing OCR ให้ความแม่นยำ (Accuracy) เฉลี่ยสูงถึง **99.7%**.
* [cite_start]**TrOCR** เป็นอีกหนึ่งโมเดลที่มีประสิทธิภาพสูงมาก โดยมีความแม่นยำในแต่ละฟิลด์ข้อมูลสูงถึง **99.87%**[cite: 81].

[cite_start]โดยสรุป **EasyOCR** ถูกเลือกใช้เป็นโมเดลหลักเนื่องจากมีความสมดุลทั้งในด้านความแม่นยำและความง่ายในการติดตั้งใช้งานสำหรับเอกสารที่มีความซับซ้อน[cite: 155].
